# SOP: SaaS Vendor Selection & Contracting

**Company:** {{COMPANY_NAME}}
**Department:** IT Operations / Procurement
**Document ID:** SaaS-001

---

## 1. Purpose

Establish a standardized process for evaluating, selecting, and contracting with SaaS/cloud vendors to ensure security, compliance, cost-effectiveness, and vendor reliability.

## 2. Scope

Applies to all cloud/SaaS services including:
- Productivity tools (Google Workspace, Microsoft 365)
- Communication tools (Slack, Teams, Zoom)
- Development tools (GitHub, Jira, Vercel)
- Cloud infrastructure (AWS, Azure, GCP)
- Business applications (CRM, HRIS, Accounting)
- AI/ML services (OpenAI, Anthropic)

---

## 3. Pre-Requisition Checklist

Before initiating vendor evaluation, complete:

- [ ] Document business need
- [ ] Identify required features
- [ ] Estimate user count
- [ ] Determine budget range
- [ ] Check if existing vendor can meet need

## 7. Contract Requirements

### 7.1 Must-Have Terms

| Term | Requirement |
|------|-------------|
| Pricing | Lock in for contract term |
| Uptime SLA | ≥99.9% |
| Data ownership | Company owns all data |
| Termination | 30-day notice allowed |
| Migration | Export data anytime |
| Liability | Cap at 12 months fees |

### 7.2 Negotiation Targets

| Item | Target |
|------|--------|
| Payment terms | Net 45 or Net 60 |
| Custom terms | Review legal |
| Pilot period | 14-30 days |
| Volume discounts | 10-20% for multi-year |

## 9. Onboarding Process

After contract signed:

- [ ] Create vendor account
- [ ] Assign admin(s)
- [ ] Configure SSO (if applicable)
- [ ] Set up billing alerts
- [ ] Add to SaaS inventory
- [ ] Document in IT assets
- [ ] Schedule vendor review
