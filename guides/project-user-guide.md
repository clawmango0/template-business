# User Guide: Project Management Templates

## Overview

Project management templates help teams plan, track, and deliver projects effectively. Use these tools to maintain clarity, accountability, and visibility.

---

## Template Overview

| Template | When to Use | Project Phase |
|----------|-------------|---------------|
| Project Charter | Starting a new project | Initiation |
| Status Report | Ongoing project updates | Execution |
| Meeting Notes | Any project meeting | All phases |
| RACI Matrix | Defining roles | Planning |
| Risk Register | Identifying/managing risks | Planning + Monitoring |
| Decision Log | Recording key decisions | All phases |

---

## How to Use Each Template

### 1. Project Charter

**Purpose:** Formally authorize a project and define its scope.

**When complete:**
- [ ] Project name and description
- [ ] Key stakeholders identified
- [ ] Budget approved
- [ ] Timeline defined
- [ ] Success criteria established

**Best for:** Getting leadership buy-in before project starts

### 2. Status Report

**Purpose:** Keep stakeholders informed on progress.

**Frequency:** Weekly or bi-weekly

**Sections:**
- Accomplishments this week
- Work in progress
- Blockers/risks
- Next week's plan
- Metrics

**Tip:** Keep it one page maximum

### 3. Meeting Notes

**Purpose:** Document decisions and action items.

**Always include:**
- Date and attendees
- Key discussion points
- Decisions made
- Action items with owners and dates

**Best practice:** Send within 24 hours of meeting

### 4. RACI Matrix

**Purpose:** Clarify roles and avoid gaps.

**How to fill:**
1. List all project tasks down the left
2. List team roles across the top
3. Assign R/A/C/I for each cell

**Rules:**
- One A (Accountable) per task
- At least one R (Responsible) per task
- Use C (Consulted) sparingly

### 5. Risk Register

**Purpose:** Proactively identify and manage risks.

**Fields to complete:**
- Risk description
- Likelihood (Low/Medium/High)
- Impact (Low/Medium/High)
- Mitigation strategy
- Owner

**Review:** Update weekly, especially after changes

### 6. Decision Log

**Purpose:** Create audit trail for decisions.

**For each decision, record:**
- Context/background
- Options considered
- Final decision
- Rationale
- Owner

## Best Practices

### Starting a Project

```
1. Complete Project Charter → Get approval
2. Create RACI → Assign roles
3. Identify risks → Create register
4. Plan meetings → Set recurring notes
5. Establish status → Set weekly reports
```

### Running Effective Meetings

1. **Before:** Send agenda in meeting notes template
2. **During:** Assign note-taker, track action items
3. **After:** Send notes within 24 hours
4. **Follow-up:** Track action items to completion

### Communication Cadence

| Audience | Frequency | Format |
|----------|-----------|--------|
| Team | Daily | Standup |
| Manager | Weekly | Status report |
| Stakeholders | Bi-weekly | Status report |
| Executive | Monthly | Summary + risks |
| Sponsor | Milestones | Charter review |

---

## Common Mistakes

| Mistake | Solution |
|---------|----------|
| No charter | Scope creep → Define first |
| Unclear roles | Use RACI |
| No tracking | Log every decision |
| Stale risks | Review and update weekly |
| No follow-through | Assign owners + deadlines |
